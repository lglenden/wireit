Execute Perspective Layout

Above is the workflow execution perspective page. In this perspective, analysts can execute the workflow to observe the analysis results real-time. (Note that this page does not include information about the layout outside the execute perspective; The perspective comprises the following sections:

  1. List of open workflow tabs. This list can be expanded and collapsed if the analyst wants more room to see the workflow execution.
  2. Execution Tool bar. Set of button actions such as “Mark Viewpoint”, “Run”, “Step”, “Step-in” , “Step- out”, “Set Breakpoint”, “Clear All Breakpoints”, “Run to Breakpoint”, “Pause” and “Terminate”.
  3. Workflow Execution Canvas.. Place where an analyst can see the graph model of the workflow and it’s execution progress with services blinking as the workflow progresses through the execution.
  4. Logger Console. All the execution errors and execution progress is shown in this logger console. Analyst can expand and collapse this console.
  5. Intermediate Results Console As the workflow execution progresses, all the intermediate results of the workflow for the services that are marked with a view point will be displayed here.
