Workflow Analyses

Here, the user has various analyses available to use. These analyses allows user to analyze various properties of the workflow. The tool currently has one example analyses available to analyze different properties of the workflow based on historic data. As, more analyses become available they will be displayed in the analyses list for the user to chose from.

Workflow Semantic Analyses using Bigram Algorithm

Above is the example screenshot of what these analyses look like in the tool. This analysis uses bigrams (see Wikipedia for more information) to analyze your workflow and make suggestions.

These analyses use the historical data stored on SORASCS to find patterns in this past data. These patterns can then be applied to the workflow that you chose through different analyses.

Currently there are three types of analysis available using bigrams. You can select which one you want below.

  • Order Analysis: This looks at the ordering of your workflow and checks it against the historical data. If there are services that make more sense in another order, they are suggested.
  • Next Service Analysis: This suggests a next service for the end of your workflow, based on the historical data. It can be helpful if you are not sure how to proceed with your workflow construction.
  • Previous Service Analysis: . This suggests a previous service for the beginning of your workflow, based on the historical data. It can be helpful if you know 'where you want to be' but aren't sure how to get there.
