Analyze Perspective Layout

Above is the workflow analyses perspective page. In this perspective, users can analyze semantic and/or other various properties of the workflow, for example, performance, security given the available analyses. (Note that this page does not include information about the layout outside the analyze perspective; The perspective is broken down into five sections described below:

  1. List of open workflow tabs. This list can be expanded and collapsed if the user wants more room to see the workflow that is chosen to be anlayzed.
  2. Available Analyses. This is the list of available analyses that the user can choose from to analyze various properties of the workflow.
  3. Analysis to Run. Once the analyses is chosen, the description of what the analyses entails and steps to take to further run the analyses are shown over here.
  4. Workflow Model. Here the graph model of the workflow being analyzed will be displayed.
  5. Results. When the user runs the analyses, the results will be displayed over here. User can expand or collapse this console.
