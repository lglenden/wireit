Workflow Validation

This page details syntactic validation of a workflow. This is different from semantic analysis that is discussed in a different section of the user manual. Syntactic validation is necessary in order for the workflow to execute.

Generally speaking, there are two types of syntactic checks in the tool. Some of them are dynamic, which means that they happen instantly during the construction of a workflow. Others can only be checked when you explcitily validate the workflow.

Dynamic syntactic checks

Dynamic syntactic checks are those that happen instantly during the construction of a workflow. They are intended to give you immediate feedback so that you can quickly correct these problems. Current dynamic syntactic checks include:

  • Ports type check. This checks the type of the ports between two services being connected. These types should match; otherwise this action is not allowed.
  • Service configuration parameter type check. This checks the type of the value for a configuration parameter that is entered. Invalid values may not be entered.
  • Self connection. This checks for whether you're trying to connect two ports on the same service to each other.
  • Single connection between two ports. This checks that only one connection (wire) is allowed between two ports.
  • Input-Input and Output-Output. This checks that all connections are between only one input and only one output.

Syntactic validation

You can syntactically validate a workflow in its entirety by clicking the "Validate" button in the compose perspective. This runs all syntactic checks against the currently opened workflow and notifies you of any problems. In addition to the checks listed in the previous section, this validation also includes:

  • Fork service connection. This checks that a fork service can be connected to as many data services as desired, but at most one of a normal service or exposed.
  • Fork service exposed. This checks that a fork service cannot be exposed on its input port.
  • Required ports. This checks that all ports marked as "required" either have a connection or are exposed.
  • Required configuration parameters. This checks that all configuration parameters marked as "required" either have a value, UI service connection, or are exposed.
