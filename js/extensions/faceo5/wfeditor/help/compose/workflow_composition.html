Workflow Composition

Workflow composition is one of the most basic uses of the workflow management tool. Composition consists of orchestrating a workflow from a number of connected services. This page describes how to create, load, save, and edit a workflow.

Create a new workflow

When the tool first loads, the initial state is a new workflow. You can immediately begin composing a new workflow from this screen.

When you have multiple workflows open, you can open a new workflow by clicking the "New" button at the top of the perspective. This will create a new workflow tab with a blank workflow. Before giving the workflow a name explicitly, its name is set to "New Workflow."

Save a workflow

When you have one or more workflows open, you must save them before you perform many features of the workflow tool, including loading, validating, and executing. There are a number of options for saving:

  • Save. This option saves the workflow under your user account with the name and properties that you have specified. If you haven't specified a name, you will be prompted for one. If you give a name that is already associated with a workflow, the old value is overwritten.
  • Save As. This option prompts you for a new name for your workflow and then saves it with that name and the current workflow properties that have been set.
  • Save All. This option is like "save" but it applies it to all workflows that are currently opened in the tool.

Load a workflow

From the initial screen, or any time afterward, you can always load an existing workflow that you have created and saved in the past. To do this, click the "Open" button or one of the options in its submenu (explained below). A screen will appear to prompt you for which workflow you want to open. To pick a workflow to open, click on the name of the one that you want. The workflow will then be loaded into the appropriate tab and the perspective will change to reflect its information.

There are a number of options for loading:

  • Open. This will prompt you for which workflow you want to load in a new tab; in other words, it leaves all other open workflows alone. However, if the currently opened tab is a new workflow, it will be opened in the current tab.
  • Open and Replace. This will again prompt you for which workflow you want to load, but the difference is that it replaces whatever workflow you currently have opened with the one that you have selected.

Close a workflow

When one or more workflows is currently open in the tool, they can be closed. This only closes them from the tool and does not remove the saved workflow. There are a number of options for closing:

  • Close. This option closes the current workflow and the currently opened tab.
  • Close and Replace. This option closes the current opened workflow and replaces the current tab with a new workflow.
  • Close all. This option closes all currently opened workflows in the tool.

Delete a workflow

When one or more workflows is currently open in the tool, they can be deleted. This both closes the workflow in the tool and removes the saved workflow. This action cannot be undone, so use it with care! There are a number of options for deleting:

  • Delete. This option deletes the current workflow and closes the currently opened tab.
  • Delete and Replace. This option deletes the current workflow and replaces the current tab with a new workflow.

Undo and redo actions

In the composition perspective, you will see an "undo" and a "redo" button at the top of the perspective. The tool has an undo/redo event system only for select composition events. The buttons will be enabled and disabled at the appropriate times to let you know when there are actions available to undo and redo. There is a limit on the size of the undo/redo stacks, and performing any additional actions will invalidate any actions in the "redo" stack.

Currently, the following events can be undone/redone:

  • Adding a service to the canvas
  • Removing a service from the canvas
  • Moving a service on the canvas
  • Adding a connection (wire) to two services on the canvas
  • Removing a connection (wire) from two services on the canvas
  • Changing the value of a service's configuration parameter
  • Changing the workflow meta-information
  • Adding/removing dynamic ports on a fork or UI service
