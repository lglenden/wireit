Data Service Configuration

Data services will allow the user to access a pre-existent set of files stored on SORASCS and also provide opportunity to create new data sets that can be used in the different workflows.

Using data on SORASCS.

Data services offered by SORASCS will be organized on two leves:

  1. Project: The data will be grouped in a project that will contain data related to some specific topic, for example: data related to Sudan events.
  2. Categories: The data inside the projects will be organized depending on the nature of the data, for example: reports, thesauri, lists, etc.

Aside from this organization, data services can be either input or output, if the user has permission (configured in SORASCS), he or she will be able to change the direction of the data service, there will be a small icon next to the data service in the canvas to switch between input or output. By performing this action, any wires that may be connected to the data service will disconnect.

Uploading local data to SORASCS.

As we can see in the right section of the screen, there's the properties column that contains minimap, service properties, workflow properties and data properties, let's review data properties.

This section will allow the user to create new data sources (input or output) that will be available as any othere data services that is already in SORASCS. Let's proceed with the creation of output data.

Configuring data services.

Output data.

First, we activate the output option and we will notice that one of the fields (File(s)) disappears, since we won't upload any data this is expected. We proceed with the setting of the next properties:

  • Name: The name that the data will have.
  • Project: The project (as explained before) the data will be part of (Sudan data, Russia espionage, World domination, etc)
  • Category: The category will allow the user to set the type of information that this output will be, for example: thesauri, list or any other category that the user wants to create, we can set any amount of categories to the data just by clicking on the plus and minus symbol below the category input.

Finally, we press the create button to create the new output data service.

Input data.

First, we activate the input option and proceed with the setting of the next properties.

  • Name: The name that that the data will have.
  • File: The files that will be part of this data service will be set by the user, the user can browse and select any file in his or her computer, the user cannot select directories,
    • Sending multiple files: The only way to send multiple files is to zip them and then selecting it.
    • Infinite amount of files: The user can add as many files as needed, by pressing the plus button below the file field, we can add more fields and by pressing the minus button we remove them.
  • Project: The project (as explained before) the data will be part of (Sudan data, Russia espionage, World domination, etc)
  • Category: The category will allow the user to set the type of information that this output will be, for example: thesauri, list or any other category that the user wants to create, we can set any amount of categories to the data just by clicking on the plus and minus symbol below the category input.

After setting these properties, we create the data input service by clicking on the create button. Once the data is on SORASCS, a new element will appear in the palette making it available to use it like any other data service. The type of the new data service is provided by SORASCS after it has been created.
